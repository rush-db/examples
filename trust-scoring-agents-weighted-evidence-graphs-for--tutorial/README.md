# Trust Scoring Agents: Weighted Evidence Graphs for Response Reliability

## Overview

This project demonstrates how to build a **Trust Scoring system** for AI agents using RushDB's property graph capabilities. By modeling evidence as weighted graph edges, we can calculate response reliability scores that factor in source quality, corroboration, and agent track records.

## What It Demonstrates

- **Graph-based trust modeling**: Represent agents, responses, evidence, claims, and sources as interconnected records
- **Weighted evidence aggregation**: Calculate trust scores by traversing evidence chains and aggregating weights
- **Source reliability scoring**: Track and score information sources based on historical accuracy
- **Agent track records**: Maintain historical trust scores for agents to inform future scoring
- **Claim verification**: Map claims to evidence and score claim reliability

## Core Concepts

### Data Model

| Label | Description |
|-------|-------------|
| `AGENT` | An AI agent that generates responses |
| `RESPONSE` | A response generated by an agent |
| `EVIDENCE` | A piece of supporting or contradicting evidence |
| `CLAIM` | A specific claim made within evidence |
| `SOURCE` | The origin of evidence (article, database, etc.) |

### Relationships

| Type | From → To | Description |
|------|-----------|-------------|
| `GENERATED` | AGENT → RESPONSE | Agent authored the response |
| `SUPPORTS` | EVIDENCE → RESPONSE | Evidence supports the response |
| `CONTRADICTS` | EVIDENCE → RESPONSE | Evidence contradicts the response |
| `MAKES` | CLAIM → EVIDENCE | Claim is made in evidence |
| `CITES` | CLAIM → SOURCE | Claim cites a source |
| `VOUCHES_FOR` | AGENT → AGENT | One agent vouches for another |

### Trust Score Algorithm

```
Response Trust = Σ(Evidence Weight × Source Reliability × Corroboration Factor)

where:
- Evidence Weight: Base weight (0-1) assigned to evidence
- Source Reliability: Historical accuracy of the source (0-1)
- Corroboration Factor: Number of supporting vs contradicting evidence
```

## Setup

### Prerequisites

- Python 3.10+
- A RushDB account (free tier works)

### Installation

```bash
# Install dependencies
pip install -r requirements.txt

# Configure environment
cp .env.example .env
# Edit .env and add your RUSHDB_API_KEY
```

### Get Your RushDB API Key

1. Sign up at https://rushdb.com
2. Create a new project
3. Copy your API key to `.env`

## Running the Example

```bash
# Seed mock data (creates agents, responses, evidence, etc.)
python seed.py

# Run the trust scoring demonstration
python main.py
```

The `seed.py` script is idempotent — it checks for existing data before creating new records. Run it multiple times safely.

## Key RushDB Features Used

- **Records & Labels**: All entities modeled as typed records
- **Relationships**: Graph edges connecting evidence, claims, sources, and agents
- **Transactions**: Atomic creation of complex graph structures
- **Query with filters**: Finding responses by trust score threshold
- **Relationship traversal**: Following evidence chains to calculate scores

## Why RushDB for Trust Scoring?

1. **Native graph model**: Relationships are first-class citizens, not foreign keys
2. **Schema flexibility**: Add new evidence types without migrations
3. **Vector support**: Enable semantic search over evidence and claims
4. **ACID transactions**: Ensure consistency when creating complex graph structures
