# RushDB 实战：从零构建向量+图谱混合检索系统

## 项目概述

本项目演示如何使用 RushDB 构建一个开发者文档智能搜索系统，同时支持：

- **向量语义搜索**：通过自然语言查询找到概念相似的 API 文档
- **图谱关系遍历**：查询找到的 API 被哪些服务消费、依赖哪些模块

## 传统架构 vs RushDB 架构

### 传统方案：双重数据库 + 同步逻辑

```
┌─────────────┐     ┌─────────────┐     ┌─────────────────┐
│   Vector DB │ ←→  │  Graph DB   │ ←→  │  应用代码       │
│  (Pinecone/ │     │  (Neo4j/    │     │  - 两套 SDK     │
│   pgvector) │     │   Nebula)   │     │  - 两套查询语言  │
└─────────────┘     └─────────────┘     │  - 两套事务模型  │
         ↑                ↑             │  - 数据同步问题  │
         └────────────────┴──────────────│  - 一致性风险    │
                                         └─────────────────┘
```

**痛点**：
- 需要维护两套数据模型和查询 API
- 向量数据库和图数据库之间的数据同步
- 两套事务管理，跨库操作复杂
- 运维成本翻倍（两个数据库、两套监控、两套备份）

### RushDB 方案：统一数据层

```
┌──────────────────────────────────────┐
│            RushDB                    │
│  ┌────────────────────────────────┐  │
│  │   Neo4j (统一存储层)           │  │
│  │   - 向量索引 (原生支持)        │  │
│  │   - 图关系 (原生支持)          │  │
│  │   - 单一事务模型               │  │
│  └────────────────────────────────┘  │
└──────────────────────────────────────┘
                    ↑
                    │
         ┌─────────────────┐
         │   应用代码      │
         │  - 一套 SDK     │
         │  - 一套查询语言  │
         │  - 一致性保证    │
         └─────────────────┘
```

**优势**：
- 单次写入同时包含向量和关系
- 单次查询完成向量搜索 + 关系扩展
- ACID 事务覆盖所有操作
- 一套运维体系

## 功能演示场景

### 场景 1：查询"认证机制"相关的 API 文档

1. **向量搜索**：找到与"认证机制"语义最相似的 API 文档
2. **图谱扩展**：查看该 API 被哪些服务调用、依赖哪些基础模块

### 场景 2：查询高频调用的 API

1. **关系查询**：查找所有 CONSUMED_BY 关系数量最多的 API
2. **向量补充**：对结果进行语义相似度排序

## 环境准备

### 前置要求

- Python 3.9+
- RushDB 账户（免费版即可）

### 安装依赖

```bash
pip install -r requirements.txt
```

### 配置环境变量

```bash
cp .env.example .env
# 编辑 .env，填入你的 RushDB API Key
```

## 运行演示

### 1. 生成模拟数据

```bash
python seed.py
```

这将创建一个包含以下内容的文档知识图谱：
- **API**：10+ 个开发者文档中的常见 API
- **Module**：认证、数据库、缓存等基础模块
- **Service**：使用这些 API 的业务服务

### 2. 运行主程序

```bash
python main.py
```

预期输出：
1. 向量索引创建状态
2. 语义搜索结果（"认证机制"相关 API）
3. 关系遍历结果（找到的 API 被哪些服务使用）
4. 统一查询演示（向量过滤 + 图谱扩展一次完成）

## 技术栈

| 组件 | 用途 |
|------|------|
| `rushdb>=2.0.0` | RushDB Python SDK（向量+图谱统一接口） |
| `sentence-transformers` | 本地生成文档向量嵌入（无 API Key 要求） |
| `python-dotenv` | 环境变量管理 |
